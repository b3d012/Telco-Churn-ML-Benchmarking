# Telco Churn ML Benchmarking and Model Interpretation

## Project Summary
This repository benchmarks a set of supervised machine learning models for telco churn prediction, with a focus on model comparison, PCA-based feature analysis, hyperparameter tuning, and model interpretation.

The goal is to build a recruiter-facing portfolio project that shows how different models perform on the same churn problem, what the most informative signals are, and which approach should be recommended for a real churn prediction workflow.

## Business Problem
Telecom providers want to identify customers who are likely to churn so they can prioritize retention efforts before revenue is lost. The business value is not only in predicting churn, but in understanding:

- which model performs best,
- whether dimensionality reduction helps,
- which features matter most,
- and how interpretable the final recommendation is.

## How This Project Differs From a Data-Preparation Project
This repository is intentionally not a primary data-cleaning repo.

It is designed to focus on:

- supervised model benchmarking,
- PCA and feature-space analysis,
- model tuning and comparison,
- post-model interpretation,
- and turning results into a decision-ready recommendation.

Data preparation is kept lightweight and practical so the machine learning experiments remain the main emphasis.

## Dataset Description
The project uses the Telco churn dataset stored at:

`data/raw/telco_customer_data_v2.csv`

The dataset contains customer attributes, service usage information, contract details, and a churn target suitable for binary classification.

## Machine Learning Methods Implemented
The project compares and interprets multiple approaches, including:

- logistic regression
- KNN
- decision tree
- random forest
- gradient boosting
- XGBoost benchmarking and tuning when the package is available
- PCA-based feature-space analysis
- calibration analysis
- permutation importance
- SHAP-based interpretation when available

If SHAP is unavailable, the project falls back to permutation importance for reliable model explanation.

## How to Run the Project
1. Create and activate a Python virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Open the notebooks in `notebooks/` for exploration and reporting.
4. Run the scripts in `src/` in the intended order once the analysis pipeline is implemented.

Example workflow:

```bash
python -m src.clean_and_prepare
python -m src.baseline_models
python -m src.model_benchmarking
python -m src.pca_analysis
python -m src.tune_xgboost
python -m src.evaluate_best_model
python -m src.explain_model
```

## Notes About Avoiding Target Leakage
This project is designed to avoid target leakage by:

- separating the target before any feature engineering,
- fitting preprocessing only on training data,
- keeping evaluation data untouched until final scoring,
- applying PCA and scaling within the training pipeline,
- and ensuring interpretation steps use the fitted model pipeline rather than the full dataset.

That design choice is important because churn prediction can look artificially strong if preprocessing or feature selection accidentally uses information from the full dataset.

## XGBoost Status

`xgboost` is available in this environment and the tuning run completed successfully, but the tuned model did not beat `logistic_balanced` on ROC-AUC. The benchmark winner remains the logistic model, which keeps the project honest and easy to defend in a portfolio review.
